UI 设计与前端架构文档：《销售备战教练：首战即决战》
一、 前端技术栈选型 (Antigravity 友好型)
为了让 Antigravity 能够以最少 Bug、最高还原度生成代码，系统强制采用目前最现代、最契合 AI 生成逻辑的技术栈：
●核心架构： Next.js 14 (App Router) + React 18。支持极速页面路由和原生 Server Components。
●样式引擎： Tailwind CSS。原子化类名，AI 生成界面的精准度极高。
●UI 组件库 (核心武器)： shadcn/ui。极具克制、冷峻的 B2B SaaS 高级感。绝不使用花哨的组件库。
○必用组件： Card, Button, Input, ScrollArea (滚动区), Badge (状态徽标), Tabs (用于历史库详情切换), Avatar。
●图标库： Lucide React。线条极简，完美适配 B2B 语境。
●Markdown 解析： react-markdown + remark-gfm。用于渲染 AI 输出的加粗、列表和表格。
二、 全局设计规范 (Design System)
●视觉调性： 极简、冷酷、高效。摒弃一切无意义的渐变和阴影，突出信息本身。
●色彩系统 (Color Palette)：
○主色调 (Primary)： 极简黑/深灰深蓝 bg-slate-900。
○背景色 (Background)： 全局底色 bg-slate-50，卡片及聊天区为纯白 bg-white。
○状态灯色系 (Traffic Lights)：
■🔴 未完成 / 缺失：text-rose-500 / bg-rose-100
■🟡 推测中 / 待确认：text-amber-500 / bg-amber-100
■🟢 已通过 / 就绪：text-emerald-500 / bg-emerald-100
●排版 (Typography)： 四级层级。强调粗体（Bold）在重点策略和主推武器上的应用，方便销售一眼扫中要害。

四、 给予 Antigravity 的直接生成指令 (Prompt)
将以下整段内容直接复制并喂给 Antigravity，它将直接为你生成上述三个页面的精美框架代码：

Markdown


# 角色
你是顶尖的前端架构师。请使用 Next.js 14, Tailwind CSS, shadcn/ui 和 lucide-react，为我开发一个 B2B 复杂销售 AI 辅助系统的前端 UI。

# 视觉要求
整体采用极简风格（Slate 蓝灰色系为主），背景为 `bg-slate-50`，组件底色为 `bg-white`，避免任何花哨设计，体现严谨专业的 B2B 质感。

# 页面开发任务 (共三个)

## 1. 登录页 (`/login`)
- 屏幕居中的 Card 组件。
- 包含标题“首战即决战教练中枢”，两个 Input（账号/密码），一个 Button（进入备战状态）。

## 2. 主工作台 (`/workspace`)
- 顶部导航：Logo，新建备战，历史库入口，用户头像。
- 主体采用左右 Flex 布局（w-1/3 和 w-2/3）。
- **左侧 (w-1/3)**：标题“实战备战清单”。纵向渲染 7 个项目（商机溯源、盘子与购买力、决策链与内线、痛点与武器映射、同行背书、专家匹配、价格与合同）。每个项目右侧需有一个圆形状态指示灯（使用 Tailwind 的 bg-emerald-500, bg-amber-400, bg-rose-500 表示不同状态）。
- **右侧 (w-2/3)**：高度占满剩余屏幕的 AI 对话区。包含聊天记录展示区（区分左右气泡，左侧浅灰，右侧深蓝）。底部固定一个带边框的输入区（含附件图标和发送按钮）。

## 3. 历史备战库 (`/history`)
- 主体同样采用左右布局（w-1/3 和 w-2/3）。
- **左侧 (w-1/3)**：标题“历史战役库”，带一个搜索框。下方渲染一组历史记录的列表项（包含项目名、日期、随行专家名称）。选中的列表项需要有明显的底色激活状态。
- **右侧 (w-2/3)**：顶部显示当前选中的项目标题。下方使用 shadcn/ui 的 `Tabs` 组件，包含两个 Tab 面板：“全景备战报告”和“现场交流建议与话术”。在 Tab 面板内 Mock 一段排版精美的 Markdown 文本，并在右下角放置一个“复制文档”按钮。

请输出这三个页面的完整 React 组件代码。
